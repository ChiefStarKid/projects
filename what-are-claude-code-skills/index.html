What are Claude Code skills? A plain explanation, with working examples

A practical explainer · updated 2026-07-15

If you've seen people type things like /new-thread or /code-review into Claude Code and wondered what a "skill" actually is, here's the short answer. A Claude Code skill is a reusable, packaged set of instructions that Claude reads and executes on demand. Rather than re-explaining a multi-step workflow every time you need it, you save the procedure once and invoke it by name.

So what is a skill, really?

At its simplest, a skill is a Markdown file describing a task: the steps to take, and what "done" looks like. Claude Code loads it and carries it out when you invoke it. The new-thread skill puts it plainly: it's "a .md file that Claude reads and executes as a skill." There's no compiler and no plugin binary. The instructions are the program, and Claude is the interpreter.

That means a skill is good at capturing things code alone can't:

  • Process. The right order of operations for a recurring task.
  • Judgment. What to check, and when to stop.
  • Voice and format. How the output should read, so it's consistent every time.

How is a skill different from an MCP server?

This is the most common point of confusion, so it's worth stating clearly. They operate at different layers:

MCP serverSkill
What it addsNew tools: capabilities Claude didn't have, like a database query or an API call.A repeatable workflow over the tools Claude already has.
Made ofRunning code exposing a protocol.Natural-language instructions in Markdown.
Answers"What can Claude do?""How should Claude do this recurring task?"

They're complementary. An MCP server extends what Claude can reach; a skill standardises how Claude handles a task you do over and over. Many real setups use both: tools from MCP, orchestrated by a skill.

How do you invoke one?

Two ways:

  • By name. You type a slash command like /new-thread, and Claude runs that skill's instructions.
  • Automatically. When your request matches what a skill covers, Claude invokes it for you.

For the custom-command style, setup is often just placing a Markdown file in your .claude/commands/ folder. From new-thread's own quick start: 

1. Copy new-thread.md into C:\Users\<you>\.claude\commands\
2. Type /new-thread [what you want it to do]
3. Claude reads the file and executes it

No install script, no configuration beyond dropping the file in place.

Two working examples you can read today

1. new-thread: hand a task to a fresh parallel session

new-thread solves a specific problem: you're deep in a long Claude Code session and a new, unrelated task appears. You don't want to derail your current thread, and you don't want the new one to inherit all your accumulated context. The skill reads your current session, extracts only the context the new task needs, and spins up a focused parallel session. You stay put; the offshoot runs on its own. It's a good example of a skill encoding process and judgment, not just commands.

2. link-pitch: research and write an SEO outreach pitch

link-pitch points at a client site and a target article, researches both, finds the keyword gaps the article leaves open, and produces a publisher-ready outreach email in your voice. It shows how a skill can capture a whole professional workflow, right down to a voiced deliverable, in a form you invoke with one command.

When should you write a skill?

Reach for a skill when a task is:

  • Recurring. You do it often enough that re-explaining it is a tax.
  • Multi-step with judgment. There's a right order and real decisions, not one command.
  • Consistency-sensitive. You want the same process and output shape every time.

If instead you need Claude to reach a system it currently can't, like a new API or a local app, that's a job for a tool (often an MCP server, or a thin bridge script). For the difference between those two, see MCP server vs. a direct bridge for desktop automation.

Frequently asked

What is a Claude Code skill?

A reusable, packaged set of instructions, usually a Markdown file, that Claude Code reads and executes on demand, so you invoke a saved workflow by name instead of re-typing it.

How is a skill different from an MCP server?

An MCP server adds new tools (new capabilities); a skill orchestrates the tools Claude already has into a repeatable workflow. MCP extends what Claude can do; a skill standardises how it does a recurring task.

How do you invoke a Claude Code skill?

Type a slash command like /new-thread, or let Claude invoke the matching skill automatically. Custom-command skills are Markdown files in your .claude/commands/ folder.

Are there open-source examples?

Yes. new-thread (parallel-session handoff) and link-pitch (SEO outreach) are both public and built to be read and adapted.
